| Concept                                  | Explanation                                                                                                | Key Points                                                                                                 | Example                                                                                    |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| **RAG (Retrieval Augmented Generation)** | Technique combining LLMs with a retrieval system to access external data without exceeding context limits. | Handles vast documents (e.g., enterprise 1PB data); retrieves relevant info for accurate answers.          | Querying company policies: Retrieve matching chunks from 100+ docs instead of loading all. |
| **Ingestion Pipeline**                   | Prepares docs: Chunk → Embed → Store in vector DB.                                                         | Offline process; consistent model/dims required.                                                           | 10M-token docs → 10K 1K-token chunks → Embed & store.                                      |
| **Retrieval Pipeline**                   | Query → Embed → Retrieve top similar chunks → Feed originals to LLM.                                       | Use same embedding model; retriever finds semantic matches.                                                | "Q1 sales?" → Embed query → Top 5 sales-related chunks → LLM answers.                      |
| **Context Window**                       | Maximum tokens (words/characters) an LLM processes at once.                                                | GPT-4.1: 1M tokens; Claude Sonnet: 200K; insufficient for enterprise-scale data.                           | 10M-token PDF can't fit in one go; needs chunking/retrieval.                               |
| **Token**                                | Unit of text LLM processes (e.g., word or subword).                                                        | "Hello" = 1 token; "I am" = 2 tokens; limits total input size.                                             | 1PB enterprise docs = 1.3 quintillion tokens.                                              |
| **Chunking**                             | Breaking large documents into smaller pieces for processing.                                               | Set chunk size (e.g., 1K tokens); yields manageable units (e.g., 10K chunks from 10M tokens).              | Chain-of-Thought PDF: Page 2 split into 1K-token segments (e.g., first 1K, next 1K).       |
| **Vector Embeddings**                    | Mathematical (numerical vector) representation of text capturing semantics.                                | Dimensions (e.g., 3072 for text-embedding-3-large); similar meanings = closer vectors (cosine similarity). | "Cat" [34,8,7.5,...] near "kitten" [33,8,6.9,...]; far from "elephant" [100,62,...].       |
| **Embedding Models**                     | Specialized models converting text to vectors (not LLMs).                                                  | OpenAI text-embedding-3-small/large (1536/3072 dims); Cohere, VoyageAI; higher dims = better but costlier. | Embed "cat" → fixed 3072-dim vector regardless of input length.                            |
| **Vector Database**                      | Stores vector embeddings for fast similarity search.                                                       | Pinecone, ChromaDB, FAISS, PostgreSQL; enables retrieval.                                                  | Store 10K chunk embeddings from docs for query matching.                                   |
| **Consistency**                          | Use identical embedding model/dimensions for docs and queries.                                             | Mismatch = poor retrieval (like different languages); re-embed if changing.                                | text-embedding-3-large (1536 dims) for both docs and queries.                              |
