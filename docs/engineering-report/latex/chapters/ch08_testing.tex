\chapter{Testing and Validation}
\label{ch:testing}

Verification system for the API-first handoff: strategy, unit catalog, Compose-backed verify scripts, sample-query matrix, requirements traceability, and honest gaps. Pass/fail counts are not invented---screenshot placeholders must be filled from real Compose runs before final submission.

\section{Test strategy}

\begin{enumerate}[leftmargin=*]
  \item \textbf{Unit tests} (\texttt{tests/}) --- fast; no live Neo4j/LLM for most cases; guardrails, intent, mood gate, memory keying, ingest helpers.
  \item \textbf{Verification scripts} (\texttt{scripts/verify\_*.py}) --- live Compose stack; contract smoke, retrieval isolation, mood, guardrails, question-form regression.
  \item \textbf{Manual / UI checks} --- test page journeys J2--J5; Neo4j Browser; \texttt{/health} and Swagger.
\end{enumerate}

\section{Results and gaps}

\begin{itemize}[leftmargin=*]
  \item \textbf{Automated design:} unit + verify suites above are the authoritative regression harness.
  \item \textbf{Documented logs:} \texttt{docs/testing/Results.md} and \texttt{Test\_Cases.md} remain templates---not a substitute for pasted run evidence.
  \item \textbf{This report:} Figures~\ref{fig:pytest} and~\ref{fig:verify} are slots for real outputs after a Compose validation pass. Until filled, do not claim a numeric pass rate in the Executive Summary.
\end{itemize}

\noindent
\textbf{Known gaps:} no CI workflow files in the repository (local/Compose verification is the Implemented path); formal \texttt{docs/testing/*} case/result tables are still stubs; end-to-end latency and scale NFRs remain \textbf{design targets} unless measured evidence is attached; verify scripts require a healthy stack, ingested \texttt{demo} data, and a valid LLM key for non-templated paths.
