\section{Product contract and data lock}

PRD user-story acceptance~\cite{prd} maps onto the FRs above: US-1 holdings Q\&A (user-scoped Cypher; refuse; citations); US-2 classifier mood with confidence gating; US-3 stable API; US-4 no advice / no hallucination; US-5 dual memory (clearing Redis does not wipe the portfolio graph); US-6 Compose handoff with idempotent ingest. \textbf{Planned}: semantic cache; streaming; LLM I/O observability (e.g.\ Langfuse); horizontal API scale. \textbf{Future Scope}: host SSO/JWT; richer entities (transactions, historical snapshots); managed Neo4j/Redis clustering~\cite{prd}.

Data lock~\cite{prd}: legacy seed snapshot \texttt{backend/portfolio\_data (27).json} (PRD canonical shape); operators should ingest the normalised per-user files under \texttt{backend/portfolios/\{user\_id\}\_portfolio.json} (\texttt{demo}, \texttt{u\_alpha}, \texttt{u\_beta}). Prices and P\&L are snapshot values at ingest time, not live market values.

\section{Architecture intent}

TRD principles lock the engineering contract~\cite{trd}: stateless API; \texttt{user\_id} as security anchor (every graph read from \texttt{(:User \{id: \$user\_id\})}; no unscoped MATCH); LLM = language only; one retrieval, one LLM call; ingested snapshot only. Stack, pins, env catalogue, and Compose topology are recorded as-built in Chapters~\ref{ch:architecture}--\ref{ch:deployment}. Explicitly out of MVP: PostgreSQL/second primary DB, LangChain/LlamaIndex agent loops, vector-DB RAG as primary retrieval, live market APIs. Managed Neo4j/Redis is \textbf{Future Scope}.
