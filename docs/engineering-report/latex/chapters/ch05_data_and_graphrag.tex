\section{Design goals}

Data layer goals~\cite{trd,prd}: Neo4j for portfolio facts and MoodEvents; Redis for recent chat turns only; \texttt{user\_id} tenant boundary on every holdings read/write; idempotent snapshot ingest; prices and P\&L are ingest-time values, not live market state.

\noindent
\texttt{Stock} nodes are \texttt{MERGE}d globally by ticker; access still requires \texttt{(:User \{id\})-[:OWNS]->(:Portfolio)-[:HOLDS]->}. \texttt{ensure\_schema} applies idempotently at API startup and before ingest:

\begin{verbatim}
CREATE CONSTRAINT user_id IF NOT EXISTS
  FOR (u:User) REQUIRE u.id IS UNIQUE
CREATE CONSTRAINT stock_ticker IF NOT EXISTS
  FOR (s:Stock) REQUIRE s.ticker IS UNIQUE
CREATE INDEX stock_isin IF NOT EXISTS FOR (s:Stock) ON (s.isin)
CREATE INDEX mood_ts IF NOT EXISTS FOR (m:MoodEvent) ON (m.timestamp)
CREATE INDEX holding_ticker IF NOT EXISTS FOR (h:Holding) ON (h.ticker)
\end{verbatim}

\noindent
The holding-ticker index is an as-built addition for user-scoped lookups. Portfolio reads/writes that expose holdings start at the requesting user~\cite{trd}:

\begin{verbatim}
MATCH (u:User {id: $user_id})-[:OWNS]->(p:Portfolio)
// further MATCH only from u or p
\end{verbatim}

\noindent
Forbidden: global Stock/Holding scan without a path from \texttt{\$user\_id}. Mood persistence \texttt{MATCH}es an existing User$+$Portfolio and never \texttt{MERGE}s a bare User. MVP trust boundary: client-supplied \texttt{user\_id}; host JWT/SSO out of scope~\cite{prd}.

\noindent
Exemplar holding lookup (as-built in \texttt{services/retrieve.py}):

\begin{verbatim}
MATCH (u:User {id: $user_id})-[:OWNS]->(:Portfolio)
      -[:HOLDS]->(h:Holding)-[:OF]->(s:Stock)
WHERE h.ticker = $ticker OR s.ticker = $ticker
RETURN h {.*, stock_sector: s.sector, stock_industry: s.industry,
          exchange: s.exchange, isin: s.isin} AS holding
\end{verbatim}

\section{GraphRAG pipeline}

\textbf{GraphRAG}: intent $\rightarrow$ user-scoped Cypher subgraph $\rightarrow$ citations $\rightarrow$ template or one LLM call $\rightarrow$ validate---not primary vector similarity search.

\begin{table}[!htbp]
\centering
\small
\caption{Why GraphRAG fits this MVP better than primary vector RAG}
\label{tab:graphrag-vs-vector}
\begin{tabularx}{\textwidth}{@{}
  >{\raggedright\arraybackslash}p{0.22\textwidth}
  >{\raggedright\arraybackslash}p{0.35\textwidth}
  >{\raggedright\arraybackslash}X
  @{}}
\toprule
\textbf{Concern} & \textbf{Vector RAG (typical)} & \textbf{This MVP (graph)} \\
\midrule
Evidence shape & Unstructured chunks & Typed holdings / metrics / sectors \\
\addlinespace
Tenant boundary & Easy to leak via global index & Every MATCH starts at \texttt{user\_id} \\
\addlinespace
Numbers / tickers & Hard to attribute & Properties returned with node type \\
\addlinespace
Multi-hop asks & Weak without graph & Native path User$\rightarrow$Portfolio$\rightarrow$Holding$\rightarrow$Stock \\
\addlinespace
Audit & Similarity score & Citation list of node types + fields \\
\bottomrule
\end{tabularx}
\end{table}

\noindent
Principles~\cite{trd}: facts from Neo4j only; mood from classifier; LLM = optional language after evidence; refuse over invent. Intent taxonomy is in Chapter~\ref{ch:requirements}. \texttt{detect\_intent} is deterministic; \texttt{resolve\_ticker} matches the \emph{user's} universe only; \texttt{clarify\_message} short-circuits on no portfolio, unclear intent, or ambiguous/unmatched ticker (e.g.\ ``Adani Power'' $\rightarrow$ \texttt{ADANIPOWER} stays inside the security boundary). \texttt{retrieve\_subgraph} returns \texttt{found=False} without a portfolio; otherwise a minimal slice by intent:

\noindent
Holdings caps (5--15) keep prompts small. Successful retrieve builds \texttt{citations} (Portfolio/Holding/Stock/Sector + fields); refused answers clear citations. Templates use only retrieved fields. Else \texttt{generate\_grounded\_answer}: refuse if \texttt{should\_refuse\_facts}; pack \texttt{CONTEXT}/optional \texttt{MOOD}/turns/\texttt{QUESTION} via \texttt{prompts/system.txt}; \textbf{one} LLM completion; \texttt{validate\_answer}; one retry at temperature~0; then refuse. Mood is not a fact source for quantities or tickers.
